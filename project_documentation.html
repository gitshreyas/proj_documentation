Bhagavad Gita Backend

AI-Powered Chatbot for Chinna Jeeyar Swamy's Bhagavad Gita Teachings

Technical Documentation v1.0.0

Project Overview

A sophisticated RAG (Retrieval-Augmented Generation) system that provides intelligent, context-aware responses to questions about the Bhagavad Gita based on Chinna Jeeyar Swamiji's teachings and Bhagavad Ramanujacharya's Githa Bhashyam commentary.

Key Features:

  • Semantic Search: Vector-based similarity search using Gemini embeddings (3072 dimensions)
  • Conversation Memory: Session-based context retention for multi-turn conversations
  • Streaming Responses: Real-time SSE (Server-Sent Events) streaming for better UX
  • Hierarchical Chunking: Parent-child document structure for better context retrieval
  • Smart Relevance Checking: AI-powered filtering of off-topic queries
  • Source Citation: Automatic chapter, episode, verse, and YouTube link citations

Problem Statement

Challenge: Bhagavad Gita teachings from Chinna Jeeyar Swamiji are spread across multiple episodes (video format) covering different chapters and verses, making it difficult for seekers to:

  • Find specific teachings quickly without watching hours of content
  • Get contextual understanding across multiple related episodes
  • Ask follow-up questions with conversation context
  • Access teachings in a structured, searchable format

Solution: Build an AI-powered chatbot that:

  • Indexes all episode content with semantic embeddings
  • Retrieves relevant context using vector similarity search
  • Generates natural, well-cited responses using LLM
  • Maintains conversation context for follow-up questions
  • Provides source references with direct YouTube links

Design Patterns Used

Singleton Service instances (VectorService, EmbeddingService) Ensure single instance, reduce overhead

Models & APIs Used

1. Embedding Model:

Model: models/gemini-embedding-001
Dimensions: 3072
Provider: Google Gemini API
Purpose: Convert text to semantic vectors for similarity search

2. Generation Model:

Model: gemini-2.5-flash
Temperature: 0.3 (consistent formatting)
Max Tokens: 4096
Provider: Google Gemini API
Purpose: Generate natural language responses with markdown formatting

3. Vector Database:

Database: Pinecone
Index: bhagavad-gita-teachings
Environment: AWS us-east-1 (Serverless)
Metric: Cosine similarity
Dimension: 3072

Key Features & Optimizations

1. Hierarchical Chunking with Parent-Child Merging:

  • Episodes split into parent (full) and child (chunks) documents
  • During retrieval, related chunks from same parent are automatically merged
  • Provides granular search with full context

2. API Key Rotation & Rate Limit Handling:

  • Supports up to 3 Google API keys
  • Automatic rotation on rate limits (429 errors)
  • Exponential backoff retry logic
  • Daily quota tracking per key

3. Conversation Context Enhancement:

  • Detects vague queries with pronouns ("it", "this", "that")
  • Enhances search query with previous conversation context
  • Enables natural follow-up questions

4. Smart Relevance Filtering:

  • AI-powered relevance check using Gemini
  • Filters out off-topic questions gracefully
  • Context-aware (considers conversation history)

5. Streaming Response with SSE:

  • Server-Sent Events for real-time streaming
  • Chunks streamed as generated (no buffering)
  • Better UX with immediate feedback

6. Automatic Source Citation:

  • Citations woven naturally into responses
  • Dedicated sources section at end
  • YouTube links for primary sources
  • Chapter, episode, verse references
